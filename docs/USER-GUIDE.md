# JKA GardenCity Dojo Manager — User Guide

## Recommended first-use order

1. Open **Settings** and confirm dojo, training and invoice defaults.
2. Create financial account nicknames under **Expenses**.
3. Create families and guardians.
4. Create students and link them to families.
5. Open each student’s **Details** to add emergency, safety or medical information.
6. Create a term and generate Tuesday and Thursday sessions.
7. Create the current fee schedule.
8. Create charges, then record payments and invoices.
9. Record attendance.
10. Create an encrypted backup after the initial records are verified.

## People

### Students

Use **Add student** for the main register. Use **Details** for:

- linked guardians
- general or parent-shareable notes
- emergency contacts
- protected medical information
- safety alerts shown during attendance

Archive a student instead of deleting financial or attendance history.

### Families and guardians

A family groups siblings for billing and payment allocation. The primary
billing guardian appears on invoices where available.

### Enquiries and trials

Create an enquiry, update its status, schedule follow-ups and convert a
successful enquiry into a family, guardian and student record.

## Training

### Terms and sessions

Create the term dates, then select **Generate sessions** to create the normal
Tuesday and Thursday classes. Edit individual sessions for closures, special
events, instructor changes or lesson themes.

### Attendance

Select a session, mark everyone present, then change only the exceptions.
Active safety alerts are displayed without showing full medical records.

## Development

### Gradings

Create a grading event and record each student’s result. When a passed result is
approved, the student’s current belt can be updated. A grading charge can be
created at the same time.

### Progress and goals

Record ratings and technical notes, add parent-shareable summaries, set goals
and mark goals complete.

## Finance

### Fees and charges

Fee schedules are versioned. Create a new version rather than changing old fees.
Charges are confirmed financial history and should be reversed or adjusted
rather than silently rewritten after payments exist.

### Payments and invoices

A family payment can be split across siblings. Any unallocated portion remains
family credit. Use **Receipt** to print or save a payment receipt. Invoices are
created from existing confirmed charges.

### Expenses

Create suppliers and account nicknames, record expenses, add expense payments
and maintain recurring-expense schedules.

### Banking

Import a reviewed Kiwibank CSV. Imported rows remain pending until confirmed,
rejected or matched. Use reconciliation to compare the statement closing
balance with confirmed movements.

Never store online-banking passwords, PINs, card numbers or security codes.

## Records

### Reports

Choose a report, apply the available filters, then print or export CSV.

### Communication

Record calls, emails, discussions and payment reminders. Create follow-up tasks
for items that still require action.

## System

### Backup and Sync

Create a password-encrypted backup and store it securely outside public GitHub.
The password is not stored and cannot be recovered. Restore uses merge mode and
does not intentionally overwrite existing IDs.

### Audit History

Review recorded changes, recover eligible soft-deleted records and review
registered devices or sync conflicts.

### Settings

Manage dojo defaults and authorised Microsoft accounts. The app prevents the
last active Administrator from being removed.

## Install the app

Use the in-app **Install app** button. In Microsoft Edge, the manual path is:

**Menu → Apps → Install JKA GardenCity Dojo Manager**
